# 量子计算系统入门课程：概念、原理、练习与答案

这是一份面向初学者的系统入门文档。它按照一条比较自然的学习路线组织：

1. bit、qubit、叠加、测量
2. 常见量子门：`X`、`Z`、`H`、`CX`
3. Bell 态和纠缠
4. 量子电路模型
5. 用 Qiskit 跑 superposition、Bell、teleportation
6. Grover 和 QPE
7. VQE、QAOA 这类混合量子-经典算法
8. 线性代数、复数、张量积、特征值
9. 真实硬件、噪声、纠错

建议学习方式：

- 第一遍：先读概念和例子，不纠结所有数学细节。
- 第二遍：做练习题，再看参考答案。
- 第三遍：用 Qiskit 把关键例子跑出来，改参数观察结果。

## 第 1 章：Bit、Qubit、叠加、测量

### 学习目标

学完本章，你应该能解释：

- bit 和 qubit 的区别。
- 为什么 qubit 可以写成 `α|0⟩ + β|1⟩`。
- 振幅和概率的关系。
- 为什么测量会把量子态变成 classical 结果。
- 为什么量子计算不是“同时试完所有答案然后直接读出来”。

### 1.1 经典 bit

经典计算的基本信息单位是 bit。一个 bit 只能取两个值之一：

```text
0 或 1
```

多个 bit 组成 bitstring，例如：

```text
001
1011
110010
```

经典计算机的状态通常可以理解为某个确定 bitstring。比如一个 3-bit 寄存器在某一时刻可能是：

```text
101
```

它不是 20% 是 `001`、30% 是 `010`、50% 是 `101`，而是确定地处于某个状态。

### 1.2 Qubit

量子计算的基本信息单位是 qubit。一个 qubit 的一般状态写作：

```text
|ψ⟩ = α|0⟩ + β|1⟩
```

其中：

- `|0⟩` 和 `|1⟩` 是两个计算基态。
- `α` 和 `β` 是复数，称为振幅。
- 必须满足归一化条件：

```text
|α|² + |β|² = 1
```

测量时：

```text
测到 0 的概率 = |α|²
测到 1 的概率 = |β|²
```

### 1.3 叠加

如果一个 qubit 处于：

```text
(|0⟩ + |1⟩) / √2
```

那么测量 `0` 和 `1` 的概率都是：

```text
1/2
```

这个状态叫叠加态。它不是经典意义上的“要么 0 要么 1，只是我们不知道”；它是一个可以发生干涉的量子状态。

一个特别重要的区别：

```text
经典随机混合：只是我们不知道系统到底是 0 还是 1。
量子叠加态：0 和 1 的振幅之间保留相位关系，可以参与干涉。
```

### 1.4 振幅不是概率

振幅可以为负，也可以是复数。概率不能为负，也不能是复数。

例如两个状态：

```text
(|0⟩ + |1⟩) / √2
(|0⟩ - |1⟩) / √2
```

直接测量时，它们都是：

```text
0 的概率 = 1/2
1 的概率 = 1/2
```

但它们并不是同一个量子态，因为相对相位不同。后续再接量子门时，它们会发生不同的干涉结果。

这就是量子算法的核心：

```text
不是直接操纵概率，而是操纵振幅和相位，让正确答案更容易被测到。
```

### 1.5 测量

测量会把量子态变成 classical 结果。

例如：

```text
|ψ⟩ = α|0⟩ + β|1⟩
```

测量后，只能得到：

```text
0 或 1
```

得到结果后，原来的叠加态通常会坍缩到测量结果对应的基态。

所以量子程序常见流程是：

```text
准备初态
施加一系列量子门
让振幅发生干涉
最后测量
统计很多次测量结果
```

### 1.6 与经典计算的异同

| 维度 | 经典计算 | 量子计算 |
| --- | --- | --- |
| 信息单位 | bit | qubit |
| 基本状态 | `0` 或 `1` | `α|0⟩ + β|1⟩` |
| 运算对象 | 逻辑值 | 振幅向量 |
| 中间读取 | 通常可以读取 | 测量会扰动态 |
| 复制状态 | 可以复制 bit | 未知量子态不可克隆 |
| 典型机制 | 布尔逻辑 | 叠加、相位、纠缠、干涉 |
| 输出 | 确定值或经典随机值 | 测量样本和概率分布 |

### 1.7 常见误区

误区 1：qubit 同时等于 0 和 1，所以可以直接并行算所有答案。

更准确的说法：qubit 的状态是基态的线性组合。你不能直接把所有分支都读出来，只能通过设计干涉过程提高目标答案的测量概率。

误区 2：叠加就是普通随机。

普通随机没有可控相位关系；量子叠加有相位关系，因此能干涉。

误区 3：测量只是“看一眼”。

测量是物理过程，会把量子态转成 classical 结果，并通常破坏原来的叠加态。

### 本章练习

1. 如果 `|ψ⟩ = √3/2 |0⟩ + 1/2 |1⟩`，测到 `0` 和 `1` 的概率分别是多少？
2. 判断 `1/2|0⟩ + 1/2|1⟩` 是否是合法量子态。
3. 判断 `1/√2|0⟩ - 1/√2|1⟩` 是否是合法量子态。
4. 为什么单次测量不能证明一个 qubit 处于叠加态？
5. `(|0⟩ + |1⟩)/√2` 和 `(|0⟩ - |1⟩)/√2` 直接测量的概率相同吗？它们是否是同一个状态？
6. 用一句话解释“振幅”和“概率”的区别。
7. 为什么说量子计算不是普通意义上的并行计算？
8. 如果某个 qubit 测量 `0` 的概率是 0.36，测量 `1` 的概率是多少？

### 本章参考答案

1. `P(0)=3/4`，`P(1)=1/4`。
2. 不是。因为概率和是 `|1/2|² + |1/2|² = 1/4 + 1/4 = 1/2`，不等于 1。
3. 是。概率和是 `1/2 + 1/2 = 1`。
4. 单次测量只给一个 classical 结果，比如 0 或 1；叠加态的信息体现在重复实验的概率分布和后续干涉中。
5. 直接测量概率相同，都是 50/50；但不是同一个状态，因为相对相位不同。
6. 振幅是量子态中的复数系数，概率是振幅模平方后的实数。
7. 因为即使状态包含多个基态振幅，测量也只能得到一个结果；算法必须通过干涉让想要的结果概率变大。
8. `P(1)=0.64`。

## 第 2 章：常见量子门 X、Z、H、CX

### 学习目标

学完本章，你应该能解释：

- 为什么量子门必须是可逆的酉变换。
- `X`、`Z`、`H`、`CX` 分别做什么。
- 为什么 `Z` 门改变相位但不直接改变测量概率。
- 为什么 `H` 门既能制造叠加，也能把相位差变成概率差。
- `CX` 如何制造纠缠。

### 2.1 量子门是什么

量子门是作用在量子态上的线性变换。单 qubit 状态可以写成向量：

```text
|ψ⟩ = [α, β]^T
```

量子门就是矩阵：

```text
新状态 = 门矩阵 × 原状态
```

为了保持总概率为 1，量子门必须是酉矩阵。直观上，酉矩阵不会改变向量长度。

### 2.2 X 门

`X` 门类似经典 NOT：

```text
X|0⟩ = |1⟩
X|1⟩ = |0⟩
```

矩阵：

```text
X = [0 1]
    [1 0]
```

对一般态：

```text
X(α|0⟩ + β|1⟩) = β|0⟩ + α|1⟩
```

直觉：交换 `|0⟩` 和 `|1⟩` 的振幅。

### 2.3 Z 门

`Z` 门改变相位：

```text
Z|0⟩ = |0⟩
Z|1⟩ = -|1⟩
```

矩阵：

```text
Z = [1  0]
    [0 -1]
```

对一般态：

```text
Z(α|0⟩ + β|1⟩) = α|0⟩ - β|1⟩
```

如果直接测量，`β` 和 `-β` 的模平方相同，所以测量概率不变。但后续门可能把这个相位差转换成可测概率差。

### 2.4 H 门

Hadamard 门：

```text
H|0⟩ = (|0⟩ + |1⟩)/√2
H|1⟩ = (|0⟩ - |1⟩)/√2
```

矩阵：

```text
H = 1/√2 [1  1]
         [1 -1]
```

它有两个特别重要的用途：

1. 从 `|0⟩` 制造均匀叠加。
2. 把相位差变成测量概率差。

例如：

```text
H (|0⟩ + |1⟩)/√2 = |0⟩
H (|0⟩ - |1⟩)/√2 = |1⟩
```

这说明相位虽然不能直接测量，但可以通过后续门转成可测结果。

### 2.5 CX 门

`CX` 也叫 CNOT，是最重要的双 qubit 门之一。

规则：

```text
控制位为 1 时，翻转目标位。
控制位为 0 时，不做事。
```

假设第一个 qubit 是控制位，第二个 qubit 是目标位：

```text
CX|00⟩ = |00⟩
CX|01⟩ = |01⟩
CX|10⟩ = |11⟩
CX|11⟩ = |10⟩
```

如果控制位处于叠加，`CX` 可以制造纠缠：

```text
|00⟩
-- H on first qubit -->
(|00⟩ + |10⟩)/√2
-- CX -->
(|00⟩ + |11⟩)/√2
```

### 2.6 为什么量子门必须可逆

经典门 `AND`：

```text
0 AND 0 = 0
0 AND 1 = 0
1 AND 0 = 0
1 AND 1 = 1
```

从输出 `0` 无法知道输入是哪一种，所以不可逆。

量子演化在封闭系统中必须是可逆的，因此不能直接使用普通 `AND` 作为量子门。但可以用可逆门，例如 Toffoli，把经典逻辑嵌入量子电路。

### 本章练习

1. 计算 `X|0⟩` 和 `X|1⟩`。
2. 计算 `Z|0⟩` 和 `Z|1⟩`。
3. 计算 `H|0⟩` 后测量 `0/1` 的概率。
4. 计算 `H|1⟩` 后测量 `0/1` 的概率。
5. 证明 `H(H|0⟩) = |0⟩`。
6. 对 `(|0⟩ + |1⟩)/√2` 施加 `Z` 后，再施加 `H`，最终状态是什么？
7. `Z` 门为什么不改变直接测量概率？
8. 对 `|00⟩` 先对第一个 qubit 施加 `H`，再施加 `CX`，最终状态是什么？
9. 如果输入是 `|11⟩`，`CX` 输出是什么？
10. 为什么经典 `AND` 门不能直接作为量子门？

### 本章参考答案

1. `X|0⟩=|1⟩`，`X|1⟩=|0⟩`。
2. `Z|0⟩=|0⟩`，`Z|1⟩=-|1⟩`。
3. `H|0⟩=(|0⟩+|1⟩)/√2`，测量 0/1 各 1/2。
4. `H|1⟩=(|0⟩-|1⟩)/√2`，测量 0/1 各 1/2。
5. `H|0⟩=(|0⟩+|1⟩)/√2`，再施加 H 得到 `(H|0⟩+H|1⟩)/√2 = |0⟩`。
6. 先变成 `(|0⟩-|1⟩)/√2`，再施加 H 得到 `|1⟩`。
7. 因为 `Z` 只把 `|1⟩` 的振幅乘以 `-1`，模平方不变。
8. `( |00⟩ + |11⟩ ) / √2`。
9. `|10⟩`，因为控制位是 1，所以目标位从 1 翻成 0。
10. 因为 `AND` 不可逆，多个输入会映射到同一个输出；量子门必须可逆。

## 第 3 章：Bell 态与纠缠

### 学习目标

学完本章，你应该能解释：

- 两 qubit 的四个计算基态是什么意思。
- Bell 态如何制备。
- 什么是纠缠态，什么是可分离态。
- 为什么纠缠不是普通经典相关性。
- 为什么纠缠不能用于超光速通信。

### 3.1 两 qubit 基态

两个 qubit 的计算基态有四个：

```text
|00⟩, |01⟩, |10⟩, |11⟩
```

含义：

```text
|00⟩ = 第一个 qubit 是 0，第二个 qubit 是 0
|01⟩ = 第一个 qubit 是 0，第二个 qubit 是 1
|10⟩ = 第一个 qubit 是 1，第二个 qubit 是 0
|11⟩ = 第一个 qubit 是 1，第二个 qubit 是 1
```

一般两 qubit 态：

```text
|ψ⟩ = a|00⟩ + b|01⟩ + c|10⟩ + d|11⟩
```

归一化：

```text
|a|² + |b|² + |c|² + |d|² = 1
```

### 3.2 Bell 态

最常见的 Bell 态：

```text
|Φ+⟩ = (|00⟩ + |11⟩) / √2
```

测量结果：

```text
00 的概率 = 1/2
11 的概率 = 1/2
01 的概率 = 0
10 的概率 = 0
```

所以两个 qubit 的测量结果总是相同。

### 3.3 Bell 态制备

从 `|00⟩` 开始：

```text
|00⟩
```

对第一个 qubit 施加 `H`：

```text
(|00⟩ + |10⟩) / √2
```

再施加 `CX`，第一个 qubit 控制第二个 qubit：

```text
(|00⟩ + |11⟩) / √2
```

### 3.4 可分离态与纠缠态

如果两 qubit 状态可以写成：

```text
(α|0⟩ + β|1⟩) ⊗ (γ|0⟩ + δ|1⟩)
```

它就是可分离态。

Bell 态不能写成这种形式，所以它是纠缠态。

直觉上，纠缠态的两个 qubit 不是各自拥有独立状态，而是整体拥有一个共同状态。

### 3.5 纠缠不能超光速通信

Bell 态测量结果有强相关，但单独看 Alice 或 Bob 的结果，都是随机的：

```text
Alice 看到 0/1 各一半
Bob 看到 0/1 各一半
```

只有他们事后用 classical communication 比较结果，才会发现相关性。因此纠缠不能用来发送可控信息，也不能超光速通信。

如果想进一步理解 EPR 佯谬、贝尔不等式、薛定谔的猫和无信号定理，可以阅读 [quantum_paradoxes_zh.md](quantum_paradoxes_zh.md)。

### 本章练习

1. 写出两个 qubit 的四个计算基态。
2. 写出 `|01⟩` 的含义。
3. 对 `|00⟩` 先施加 `H` 到第一个 qubit，再施加 `CX`，得到什么状态？
4. Bell 态 `(|00⟩+|11⟩)/√2` 测量时会出现 `01` 吗？
5. 为什么 Bell 态不是两个独立 qubit？
6. 写出四个标准 Bell 态。
7. 如果 Alice 测到 Bell 态的第一个 qubit 是 1，Bob 的 qubit 会是什么？
8. 为什么纠缠不能超光速通信？
9. 判断 `(|00⟩+|01⟩)/√2` 是否纠缠。
10. 判断 `(|00⟩+|11⟩)/√2` 是否纠缠。

### 本章参考答案

1. `|00⟩`、`|01⟩`、`|10⟩`、`|11⟩`。
2. 第一个 qubit 是 0，第二个 qubit 是 1。
3. `( |00⟩ + |11⟩ ) / √2`。
4. 不会，概率为 0。
5. 因为它不能拆成两个单 qubit 状态的张量积。
6. `(|00⟩+|11⟩)/√2`，`(|00⟩-|11⟩)/√2`，`(|01⟩+|10⟩)/√2`，`(|01⟩-|10⟩)/√2`。
7. Bob 的 qubit 也是 1。
8. 因为单方测量结果不可控且随机，必须事后通过经典通信比较结果才能看到相关性。
9. 不纠缠。它等于 `|0⟩ ⊗ (|0⟩+|1⟩)/√2`。
10. 纠缠。

## 第 4 章：量子电路模型

### 学习目标

学完本章，你应该能解释：

- 量子电路由哪些部分组成。
- 为什么电路通常最后才测量。
- 为什么多个量子门组合等价于矩阵相乘。
- 什么是 `shots` 和 `counts`。
- 为什么真实硬件需要 `transpile`。

### 4.1 量子电路的基本结构

量子电路通常包含：

- qubits：量子寄存器。
- gates：量子门。
- measurements：测量。
- classical bits：保存测量结果。

典型流程：

```text
初始化 |0...0⟩
施加量子门
制造叠加、相位、纠缠和干涉
测量
统计结果
```

### 4.2 为什么通常最后测量

测量会把量子态变成 classical 结果，并通常破坏叠加和相干性。

如果太早测量，后续电路就无法继续利用：

- 相位
- 干涉
- 纠缠

所以大多数量子算法会把测量放在最后。

中途测量也不是不能用，例如量子纠错和隐形传态中会用到中途测量和 classical feed-forward，但那是有明确设计的。

### 4.3 门的组合

如果先施加 `U1`，再施加 `U2`，最后施加 `U3`，则总变换是：

```text
U = U3 U2 U1
```

电路图从左到右读，但矩阵乘法作用时右边的门先作用。

### 4.4 `shots` 和 `counts`

一次量子电路运行只得到一个测量结果。为了估计概率分布，需要重复运行很多次。

`shots` 表示重复次数。

例如：

```text
shots = 1024
counts = {"00": 520, "11": 504}
```

表示 1024 次运行中，测到 `00` 有 520 次，测到 `11` 有 504 次。

### 4.5 Transpilation

你写的电路通常是抽象电路，但真实硬件有约束：

- 支持的基础门有限。
- qubit 之间连接不是全连通。
- 不同 qubit 的错误率不同。
- 电路深度越大，越容易受噪声影响。

`transpile` 会把抽象电路编译到目标 backend 更容易执行的形式。

### 本章练习

1. 一个量子电路通常由哪些部分组成？
2. 为什么量子算法通常不在中间随意测量？
3. 如果一个电路有 3 个 qubit，它的状态向量有多少维？
4. 如果一个电路有 5 个 qubit，它的状态向量有多少维？
5. `shots=1000` 的含义是什么？
6. `counts={"0": 480, "1": 520}` 表示什么？
7. 电路中先施加 `H`，再施加 `Z`，总矩阵顺序是 `ZH` 还是 `HZ`？
8. 为什么真实硬件需要 transpile？
9. `transpile` 可能插入 `SWAP` 门，为什么？
10. 如果一个量子算法依赖相位干涉，过早测量会造成什么问题？

### 本章参考答案

1. qubits、gates、measurements、classical bits。
2. 因为测量会破坏叠加和相干性，使后续干涉失效。
3. `2^3 = 8` 维。
4. `2^5 = 32` 维。
5. 同一电路重复运行 1000 次。
6. 1000 次中测到 0 有 480 次，测到 1 有 520 次。
7. 总矩阵是 `ZH`，因为 `H` 先作用，`Z` 后作用。
8. 因为真实硬件有门集、连接拓扑、错误率等限制。
9. 因为某些需要交互的 qubit 在硬件上不相邻，需要交换状态让它们靠近。
10. 相位关系被破坏，目标答案可能无法被放大。

## 第 5 章：用 Qiskit 跑 Superposition、Bell、Teleportation

### 学习目标

学完本章，你应该能：

- 运行 superposition 和 Bell 示例。
- 理解 `AerSimulator`、`transpile`、`counts`。
- 用文字解释量子隐形传态。
- 区分真实硬件结果和模拟器结果。

### 5.1 Superposition 示例

```python
from qiskit import QuantumCircuit, transpile
from qiskit_aer import AerSimulator

qc = QuantumCircuit(1, 1)
qc.h(0)
qc.measure(0, 0)

backend = AerSimulator(seed_simulator=42)
compiled = transpile(qc, backend)
counts = backend.run(compiled, shots=1024).result().get_counts()
print(counts)
```

预期结果接近：

```text
{"0": 512, "1": 512}
```

不会每次严格一半，因为这是采样。

### 5.2 Bell 示例

```python
from qiskit import QuantumCircuit

qc = QuantumCircuit(2, 2)
qc.h(0)
qc.cx(0, 1)
qc.measure([0, 1], [0, 1])
```

预期主要出现：

```text
00 和 11
```

### 5.3 Teleportation

量子隐形传态传的是量子态，不是物体本身。

流程：

1. Alice 有未知态 `|ψ⟩`。
2. Alice 和 Bob 共享 Bell 对。
3. Alice 对自己的两个 qubit 做 Bell measurement。
4. Alice 把两个 classical bits 发给 Bob。
5. Bob 根据 bits 施加 `X`、`Z` 校正。
6. Bob 的 qubit 变成原来的 `|ψ⟩`。

关键点：

- 需要纠缠资源。
- 需要 classical communication。
- 原来的未知态被测量破坏，不违反不可克隆定理。

### 5.4 Statevector 和 counts

`counts` 是采样统计，接近真实硬件输出。

`Statevector` 是模拟器内部完整振幅，真实硬件不会直接给你。

学习时可以用 `Statevector` 看清状态变化，但理解真实运行时应主要关注测量结果。

6. 隐形传态是否复制了未知量子态？
7. `AerSimulator` 是做什么的？

2. 因为 shots 是随机采样，样本少时统计波动更大。

## 第 6 章：Grover 与 [QPE](glossary_zh.md#qpe)

### 学习目标

学完本章，你应该能解释：

- Grover 搜索解决什么问题。
- oracle 和 diffuser 分别做什么。
- 为什么 Grover 是振幅放大算法。
- [QPE](glossary_zh.md#qpe) 估计的是什么。
- [inverse QFT](glossary_zh.md#inverse-qft) 在 QPE 中起什么作用。

### 6.1 Grover 搜索

Grover 用于无结构搜索。假设有 `N` 个候选，其中一个是目标。经典搜索平均需要 `O(N)` 次检查，Grover 约需要：

```text
O(√N)
```

Grover 的步骤：

1. 准备所有候选的均匀叠加。
2. oracle 给目标态加负相位。
3. diffuser 围绕平均振幅反射。
4. 重复若干次。
5. 测量，目标态概率较高。

### 6.2 Oracle

oracle 是把问题编码进量子电路的黑盒。

Grover 的 phase oracle 通常做：

```text
目标态 |x*⟩ -> -|x*⟩
其他态 |x⟩ -> |x⟩
```

负号本身直接测不出来，但它会影响后续干涉。

### 6.3 Diffuser

diffuser 的直觉是“围绕平均振幅反射”。经过 oracle 标记后，目标态振幅和其他态振幅相位不同。diffuser 利用这个差异放大目标态。

对于 4 个候选、1 个目标，通常 1 次 Grover iteration 就可以把目标概率放大到 1。

### 6.4 [QPE](glossary_zh.md#qpe)

[QPE](glossary_zh.md#qpe) 是 Quantum Phase Estimation，目标是估计酉算子的本征相位。

如果：

```text
U|u⟩ = e^(2πiθ)|u⟩
```

[QPE](glossary_zh.md#qpe) 估计的是 `θ`。

例如：

```text
θ = 3/8 = 0.011₂
```

用 3 个 evaluation qubits，可以理想地测到：

```text
011
```

### 6.5 [QPE](glossary_zh.md#qpe) 的核心结构

[QPE](glossary_zh.md#qpe) 包括：

1. evaluation register 做 Hadamard，形成叠加。
2. 施加 controlled-`U^(2^k)`。
3. 对 evaluation register 做 [inverse QFT](glossary_zh.md#inverse-qft)。
4. 测量 phase bits。

[inverse QFT](glossary_zh.md#inverse-qft) 的作用是把相位信息转换成计算基上的 bitstring。

### 本章练习

1. Grover 解决什么类型的问题？
2. Grover 的 oracle 做什么？
3. Grover 的 diffuser 做什么？

9. `5/8` 的三位二进制是什么？

11. Grover 是否能解决所有问题？为什么？
12. QPE 和 Shor 算法有什么关系？

### 本章参考答案

1. 无结构搜索或可由 oracle 判断目标的搜索问题。
2. 给目标态加负相位，标记目标。
3. 利用振幅反射放大目标态概率。
4. 相位本身不是直接测量概率，必须通过干涉转化成概率差。
5. 通常 1 次。
6. 大约 `√16 = 4` 次量级，具体最优次数与目标数量有关。
7. `U|u⟩ = e^(2πiθ)|u⟩`。
8. `011`。
9. `101`。
10. 它把相位编码转换成可测的二进制 bitstring。
11. 不能。它只对特定 oracle 搜索问题给出平方加速，不是万能加速。
12. Shor 算法中的周期寻找可以转化为相位估计问题，QPE 是其核心子程序之一。

## 第 7 章：[VQE](glossary_zh.md#vqe) 与 [QAOA](glossary_zh.md#qaoa) 混合算法

### 学习目标

学完本章，你应该能解释：

- 什么是混合量子-经典算法。
- [VQE](glossary_zh.md#vqe) 为什么适合估计 [Hamiltonian](glossary_zh.md#hamiltonian) 低能量。
- [QAOA](glossary_zh.md#qaoa) 如何表达组合优化问题。
- ansatz、Hamiltonian、cost layer、mixer layer 的含义。
- 为什么 NISQ 阶段偏好浅电路。

### 7.1 混合量子-经典算法

当前真实量子硬件有噪声，qubit 数量和电路深度有限。因此很多实用探索采用：

```text
短量子电路 + 经典优化器
```

这种方法叫 hybrid quantum-classical algorithm。

典型流程：

```text
经典参数 θ
-> 量子电路制备状态
-> 测量得到目标函数估计
-> 经典优化器更新 θ
-> 重复
```

### 7.2 [VQE](glossary_zh.md#vqe)

[VQE](glossary_zh.md#vqe) 是 Variational Quantum Eigensolver，常用于估计 [Hamiltonian](glossary_zh.md#hamiltonian) 的低能量态。

流程：

1. 选择 ansatz，也就是带参数的量子电路。
2. 用参数 `θ` 制备状态 `|ψ(θ)⟩`。
3. 估计能量：

```text
E(θ) = ⟨ψ(θ)| H |ψ(θ)⟩
```

4. 经典优化器寻找让 `E(θ)` 更低的参数。

Hamiltonian 可以理解为描述系统能量的算子。

### 7.3 Ansatz

ansatz 是你假设的候选状态形式。它通常由参数化旋转门和纠缠门组成。

好的 ansatz 需要平衡：

- 表达能力强。
- 电路不要太深。
- 对硬件噪声相对稳健。

### 7.4 [QAOA](glossary_zh.md#qaoa)

[QAOA](glossary_zh.md#qaoa) 是 Quantum Approximate Optimization Algorithm，常用于组合优化。

它交替使用：

- cost layer：把目标函数编码成相位。
- mixer layer：让状态在不同候选解之间流动。

一层 QAOA 可写成：

```text
|ψ(γ, β)⟩ = U_M(β) U_C(γ) |+...+⟩
```

然后用经典优化器寻找好的 `γ` 和 `β`。

### 7.5 NISQ 阶段

NISQ 表示：

```text
Noisy Intermediate-Scale Quantum
```

特点：

- qubit 数量中等。
- 噪声明显。
- 没有完整容错纠错。
- 深电路很难可靠运行。

因此 VQE、QAOA 这类浅电路混合算法很受关注。

### 本章练习

1. 什么是混合量子-经典算法？
2. VQE 中 ansatz 是什么？
3. Hamiltonian 的期望值为什么可以理解为能量？
4. VQE 为什么需要经典优化器？
5. QAOA 的 cost layer 做什么？
6. QAOA 的 mixer layer 做什么？
7. QAOA 常用于哪类问题？
8. 为什么 NISQ 阶段偏好浅电路？
9. ansatz 表达能力太弱会怎样？
10. ansatz 太深会怎样？
11. VQE 和 QAOA 的共同点是什么？
12. VQE 和 QAOA 的主要区别是什么？

### 本章参考答案

1. 由量子电路估计目标函数、经典优化器更新参数的算法。
2. 带参数的候选量子电路，用来制备候选状态。
3. 在物理中 Hamiltonian 表示系统能量，态对 Hamiltonian 的期望值就是该态的平均能量。
4. 参数空间通常由经典优化器搜索，量子设备负责估计目标函数。
5. 把优化目标编码进相位或能量。
6. 让状态从一个候选解流向其他候选解，帮助搜索。
7. 组合优化，例如 MaxCut。
8. 深电路更容易受退相干和门错误影响。
9. 可能无法表示接近最优的状态。
10. 更容易被噪声破坏，运行成本更高。
11. 都使用参数化量子电路和经典优化器。
12. VQE 主要面向 Hamiltonian 低能量问题；QAOA 主要面向组合优化近似解。

## 第 8 章：线性代数、复数、张量积、特征值

### 学习目标

学完本章，你应该能解释：

- 为什么量子态用向量表示。
- 为什么量子门用矩阵表示。
- 为什么复数相位很重要。
- 多 qubit 状态如何用张量积组合。
- 特征值和 QPE 的关系。

### 8.1 向量表示量子态

单 qubit 基态：

```text
|0⟩ = [1, 0]^T
|1⟩ = [0, 1]^T
```

一般态：

```text
|ψ⟩ = α|0⟩ + β|1⟩ = [α, β]^T
```

### 8.2 矩阵表示量子门

量子门作用在状态向量上：

```text
|ψ'⟩ = U|ψ⟩
```

其中 `U` 是酉矩阵。

酉矩阵满足：

```text
U†U = I
```

直觉：酉变换保持向量长度，因此保持总概率为 1。

### 8.3 复数和相位

复数可以写成：

```text
re^(iθ)
```

其中 `θ` 是相位。

量子计算中，两个振幅的相对相位会影响干涉。即使测量概率暂时一样，后续量子门也可能把相位差变成概率差。

### 8.4 张量积

多个 qubit 的状态空间通过张量积组合。

```text
|00⟩ = |0⟩ ⊗ |0⟩
|01⟩ = |0⟩ ⊗ |1⟩
|10⟩ = |1⟩ ⊗ |0⟩
|11⟩ = |1⟩ ⊗ |1⟩
```

向量表示：

```text
|00⟩ = [1, 0, 0, 0]^T
|01⟩ = [0, 1, 0, 0]^T
|10⟩ = [0, 0, 1, 0]^T
|11⟩ = [0, 0, 0, 1]^T
```

`n` 个 qubit 的状态向量维度是：

```text
2^n
```

### 8.5 内积

内积可以表示两个状态的重叠程度：

```text
⟨φ|ψ⟩
```

如果两个状态正交，内积为 0。

例如：

```text
⟨0|1⟩ = 0
⟨0|0⟩ = 1
```

### 8.6 特征值

特征值关系：

```text
A|v⟩ = λ|v⟩
```

意思是：算子 `A` 作用在特征向量 `|v⟩` 上后，不改变方向，只乘上一个系数 `λ`。

QPE 中：

```text
U|u⟩ = e^(2πiθ)|u⟩
```

这里 `|u⟩` 是特征态，`e^(2πiθ)` 是特征值，QPE 估计的是 `θ`。

5. 10 个 qubit 的状态向量有多少维？

7. 什么是酉矩阵？
8. 为什么复数相位在量子计算里重要？

10. 用一句话解释特征值关系 `A|v⟩ = λ|v⟩`。

### 本章参考答案

1. `|0⟩=[1,0]^T`，`|1⟩=[0,1]^T`。
2. `[0,1,0,0]^T`。
3. `[0,0,1,0]^T`。

5. `2^10=1024` 维。

7. 满足 `U†U=I` 的矩阵。
8. 相对相位会影响干涉，进而影响最终测量概率。
9. 0。
10. 算子作用在某些特殊向量上时，只改变尺度或相位，不改变方向。
11. 特征值写成 `e^(2πiθ)`，QPE 估计其中的相位 `θ`。
12. 因为状态向量维度随 qubit 数量指数增长，为 `2^n`。

## 第 9 章：真实硬件、噪声、纠错

### 学习目标

学完本章，你应该能解释：

- 为什么真实量子计算机很难制造。
- 什么是退相干、T1、T2。
- 门错误和测量错误是什么。
- error mitigation 和 quantum error correction 的区别。
- 为什么一个逻辑 qubit 可能需要很多物理 qubit。

### 9.1 真实硬件的核心矛盾

量子计算需要：

```text
系统足够隔离，才能保持相干；
系统又必须足够可控，才能执行门和测量。
```

隔离和控制天然有冲突。控制会引入外界耦合，外界耦合会带来噪声和退相干。

### 9.2 退相干

退相干是量子系统和环境发生相互作用后，叠加态中的相位关系逐渐丢失。

纯叠加：

```text
(|0⟩ + |1⟩) / √2
```

退相干后更像经典混合：

```text
50% 是 |0⟩，50% 是 |1⟩
```

区别在于：前者能干涉，后者不能。

### 9.3 T1 和 T2

`T1`：能量弛豫时间。

```text
|1⟩ 衰减到 |0⟩ 的时间尺度
```

`T2`：相位退相干时间。

```text
叠加态相位关系保持的时间尺度
```

通常：

```text
T2 ≤ 2T1
```

因为能量弛豫也会破坏相位关系。

### 9.4 门错误与测量错误

门错误：你想施加某个量子门，但实际操作和理想门有偏差。

测量错误：真实状态是 `0`，读出来可能是 `1`；真实状态是 `1`，读出来可能是 `0`。

此外还有：

- 串扰
- 漏出计算空间
- 控制脉冲误差
- 热噪声
- qubit 频率漂移

### 9.5 Error mitigation

误差缓解不真正修复量子态，而是在结果层面减轻噪声影响。

例子：

- readout error mitigation
- zero-noise extrapolation
- probabilistic error cancellation

它通常需要额外实验和统计后处理。

### 9.6 Quantum error correction

量子纠错的目标是用多个物理 qubit 编码一个逻辑 qubit。

它不直接复制未知量子态，而是通过编码和 syndrome measurement 检测错误类型，再进行恢复。

量子纠错必须处理：

- bit flip 错误
- phase flip 错误
- 两者组合

真正容错量子计算通常需要大量物理 qubit 才能构造稳定的逻辑 qubit。

### 9.7 为什么硬件拓扑重要

真实芯片上，不是任意两个 qubit 都能直接执行双 qubit 门。

如果电路中需要远距离 qubit 交互，transpiler 可能插入 `SWAP` 门。`SWAP` 会增加电路深度，更多门也意味着更多错误机会。

1. 真实量子硬件的核心矛盾是什么？
2. 什么是退相干？
3. T1 描述什么？
4. T2 描述什么？
5. 为什么 T2 通常受 T1 限制？
6. 门错误和测量错误有什么区别？
7. 什么是 error mitigation？
8. 什么是 quantum error correction？
9. error mitigation 和 error correction 的核心区别是什么？
10. 为什么一个逻辑 qubit 可能需要很多物理 qubit？
11. 为什么硬件连接拓扑会影响电路执行？
12. 为什么深电路在 NISQ 设备上更难运行？

### 本章参考答案

1. 量子态需要隔离以保持相干，但又需要控制以执行计算。

3. `|1⟩` 衰减到 `|0⟩` 的时间尺度。
4. 叠加态相位关系保持的时间尺度。
5. 因为能量弛豫本身也会破坏相位关系。
6. 门错误发生在操作过程中；测量错误发生在读出 classical 结果时。
7. 在结果层面用额外实验和后处理减轻噪声影响。
8. 用多个物理 qubit 编码和保护逻辑 qubit，检测并纠正错误。
9. error mitigation 不真正保护量子态；error correction 目标是在计算过程中主动保护逻辑量子信息。
10. 因为需要冗余编码和 syndrome measurement 来检测、定位、纠正错误。
11. 不相邻 qubit 可能无法直接执行双 qubit 门，需要插入 SWAP，增加深度和错误。
12. 深电路运行时间长、门数量多，更容易受到退相干和门错误影响。

## 总复习题

1. 用一句话解释 bit 和 qubit 的区别。
2. 为什么量子计算不是简单的“并行尝试所有答案”？
3. `H` 门为什么既能制造叠加，也能读取相位差？
4. `Z` 门为什么可能不改变直接测量概率，却仍然很重要？
5. Bell 态为什么是纠缠态？
6. 为什么纠缠不能超光速通信？
7. 量子电路为什么通常最后才测量？
8. `shots` 和 `counts` 分别是什么？
9. Grover 搜索的核心机制是什么？
10. QPE 估计的到底是什么？
11. VQE 为什么叫混合量子-经典算法？
12. QAOA 的 cost layer 和 mixer layer 分别有什么作用？
13. 张量积为什么会让量子系统维度指数增长？
14. 复数相位为什么重要？
15. 退相干为什么是量子硬件的核心难题？

## 总复习题参考答案

1. bit 只能是 0 或 1；qubit 可以是 `α|0⟩+β|1⟩`，并通过测量给出概率结果。
2. 因为测量只能得到一个结果，算法必须通过干涉放大目标答案，而不是直接读出所有分支。
3. `H|0⟩` 产生叠加；同时 `H` 能把 `(|0⟩+|1⟩)/√2` 和 `(|0⟩-|1⟩)/√2` 变成不同可测状态。
4. 因为它改变相对相位，相位会影响后续干涉。
5. 因为它不能写成两个独立单 qubit 状态的张量积。
6. 因为单方测量结果随机且不可控，必须通过 classical communication 比较才能看到相关性。
7. 因为过早测量会破坏叠加、相位和纠缠，使干涉无法继续。
8. `shots` 是重复运行次数，`counts` 是各测量结果出现次数。
9. oracle 标记目标相位，diffuser 通过干涉放大目标振幅。
10. 估计酉算子特征值 `e^(2πiθ)` 中的相位 `θ`。
11. 因为量子电路负责制备和测量，经典优化器负责更新参数。
12. cost layer 编码目标函数，mixer layer 让状态在候选解之间流动。
13. 每增加一个 qubit，状态空间维度乘以 2，n 个 qubit 是 `2^n`。
14. 相对相位决定干涉效果，能影响最终测量概率。
15. 因为退相干会破坏叠加和相位关系，使量子算法依赖的干涉失效。

## 建议学习节奏

如果你每天学习 45 到 60 分钟，可以按下面安排：

| 天数 | 内容 | 目标 |
| --- | --- | --- |
| 第 1 天 | 第 1 章 | 建立 bit/qubit/测量直觉 |
| 第 2 天 | 第 2 章 | 掌握 `X/Z/H/CX` |
| 第 3 天 | 第 3 章 | 理解 Bell 态与纠缠 |
| 第 4 天 | 第 4-5 章 | 跑通 Qiskit 基础示例 |
| 第 5 天 | 第 6 章 | 理解 Grover 和 QPE |
| 第 6 天 | 第 7 章 | 理解 VQE 和 QAOA |
| 第 7 天 | 第 8-9 章 | 补数学和硬件现实 |

最有效的方法不是一次读完，而是：

```text
读一个概念 -> 跑一个小电路 -> 改一个参数 -> 解释输出变化
```

当你能用自己的话解释“振幅如何被门改变，干涉如何改变测量概率”，量子计算的主线就真正连起来了。
